# Orbit Wars：v20 vs v19 变更说明（日报用）

**日期**：2026-05-12  
**范围**：当前仓库 **`submission_v19.py`**（对照基线）与 **`submission_v20.py`**：**基础能力层**（区域价值与规划骨架）+ **v20 策略增量**；不含全盘逐行 diff。

---

## 一句话

**v19 → v20 先有一层「区域价值 + 统一边评分 + 门控」的单文件基础能力（决定后续迭代上限），v20 在此基础上增加开局剥厂、低产 defer、多源同步窗口等策略补丁，并切换默认参赛文件为 v20。**

---

## 基础能力层（区域价值评估与规划骨架）

本节对应 **「区域价值评估」模块** 及其上下游：**不是 v20 独有**——**v19 单文件里已具备同一套 inlined 结构**；两套代码都把它当作「Kaggle 单包内可演进的底座」，后续调参、加特征、接学习基本都绕不开这一层。

| 模块 / 能力 | 作用（简要） | 对后续迭代的意义 |
|-------------|----------------|------------------|
| **`RegionalGraph`** | 全图行星 **spatial 聚类**（`scipy` `fclusterdata`，目标约 4 区）、`planet_to_region`；**太阳感知**图最短路（`dijkstra` + cache），用于距离/远征成本；**`region_production`**、**`region_threat`**（本区敌星 + 邻区衰减压力） | 所有「哪片战区更值得」的量化都从这里长出来；换聚类距离、威胁公式、路径代价即改变全局扩张倾向。 |
| **`regional_capture_adjustment`** | 在 **`target_score` 之上**叠加：**同区** → 凝聚力奖励（己方区产 + 区内中立/可 snipe 敌星的 **潜力 pot**）；**跨区** → 远征税（随 ETA 放大）+ 图距离成本 | 这是 **「区域价值」进排序的主入口**：同一对 `(src,dst)` 因区属不同会系统性抬高/压低。 |
| **`capture_edge_score`** | **统一占领边评分**：`base, need, eta = target_score(...)`，再 `+ regional_capture_adjustment`；规划里 **Top-K / capture plan** 多走这一路而非裸 `target_score` | 后续加「区域 + RL 特征 + 手工项」优先接在这里，避免多套打分口径。 |
| **`target_value_in_region`** | 返回与 **`capture_edge_score` 首分量**一致的标量，给工具/兼容 | 便于离线脚本与测试对齐「你看到的区域后分」。 |
| **`ProductionTimeline` + `MultiHopPlanner`** | 简单 **产线 surplus 预测**（`predict_surplus` / `can_support_wave`）；多跳 **`Wave` 序列 scaffold**（`plan_attack_sequence`） | 当前对局内参与度可深可浅，但是 **多波次 / 分期投入** 的扩展钩；要拉高上限多在 timeline + hop 上接真 rollout。 |
| **`Snapshot.calculate_safe_surplus_v19` / `v20`** | 在 **总 `avail` 之上**，用 **`RegionalGraph.region_threat` 各 `region_threat` 推压力**，再算可_commit 战略预算 | 与 **`PlanArbiter.commit_best`** 的门控（如 `region_pressure_ratio` 等）联动，**区域态势直接约束**大开销计划。 |
| **`PlanArbiter` 持图** | 构造时注入 `regional_graph`（及 `multi_hop_planner`）；收集/评分/承诺计划时读区域上下文 | 单入口仲裁器，后续加 mode 或子规划器仍建议挂同一 `regional_graph`，避免状态分裂。 |
| **单文件自包含** | 区域与游戏状态、仲裁、仿真同文件，**无运行时依赖 `submission_*_regional.py`** | 「能上传 Kaggle 的一包」与「可维护的架构」折中；迭代时**改这一份**即生效。 |

**小结（写日报可用）**：**区域价值** = **聚类战区 + 威胁场 + 在 `capture_edge_score` 里对每条扩张边做同区/跨区价值修正** + **在 `safe_surplus` 里用全区威胁限战略投入**；**v20 在这套底座上只改策略常数与分支（下文），不改「有没有区域层」这一事实。**

## 策略与行为（主要为 v20 相对当前 v19 文件的增量）

当前仓库中的 **`submission_v19.py`** **不含**下文中的 OPENING / defer / `SYNC_ETA_WINDOW_MAX` 等补丁（以文件内是否出现对应符号为准）；**`submission_v20.py`** 在 **同一套区域基础层**上增加：

### 1. 开局单基地（solo HQ）扣血与剥厂节奏

- **v19（本仓库该文件）**：`Snapshot._reserve` 无单独「第一波剥 20 堆厂星」封顶；易出现 HQ 兵力已够 **21** 但 **`surplus` 被压在 20 以下**、晚数个回合才派兵。
- **v20**：增加 **`OPENING_FIRST_CAPTURE_SEND` / `OPENING_SOLO_HQ_RESERVE_LAST_STEP`**；在 early、仅一星、无 net_threat、非彗星等条件下，一旦 **`ships >= peel`**，在无近距离围城（`threat_horizon` 未触发高产守家加码）时把留守压到 **`threat + 1`**。敌舰已在 **8 步内**扑本星且产地 ≥3 时仍走较保守的 **`ships - peel` / 原逻辑**。

### 2. 扩张优先级：先厂后「蚊子球」

- **v20**：expand 对 **低产中立（尤其 prod≤2）** 做 **defer**（太阳安全路径上存在 **prod≥3** 厂星、且其 `capture_need` 落在 **`wait_budget`** 内则跳过近处小点）。
- **`target_score`**：**early + 中立 + prod≤1** 的 **`mite_neutral_pen`**。  
- **v19（本文件）**：无上述 defer / 蚊子球罚分。

### 3. 多源同步占领（大中立 / 胖灰星）

- **v20**：**`SYNC_ETA_WINDOW` / `SYNC_ETA_WINDOW_MAX`**，多源凑兵 **渐进放宽 ETA 带**。**v19（本文件）**无该组逻辑。
